■ 이 앱은 뭐 하는 건가요?
긴 뉴스 기사나 유튜브·SNS 글 링크를 넣으면,
사람 대신 “요약·정리”를 해 주는 투자 도우미 앱.

• 링크만 넣으면 → 핵심 내용, 분위기(긍정/중립/부정), 신뢰도 같은 느낌의 점수,
  관련이 있어 보이는 종목 이름 등을 한 화면에 모아 보여 줍니다.
• 나중에 다시 보려면 → 휴대폰(또는 브라우저) 안에 “기록”으로 쌓아 둡니다.
• 폴더에 넣거나, 메모를 달거나, 날짜·검색으로 찾을 수도 있습니다.


■ 동작을 아주 단순하게 말하면?

1) 사용자가 주소(URL)를 보냅니다.
2) 인터넷 어딘가에 있는 “분석용 컴퓨터(서버)”가 그 페이지·영상·글에서
   글자를 모읍니다. (웹이면 기사 본문, 유튜브면 자막·설명 등)
3) 그 글자를 바탕으로 AI가 요약·분석 문장을 만듭니다.
4) 결과는 사용자 기기에 저장됩니다. (지금 구조에서는 주로 내 휴대폰/브라우저 저장)


■ 이 프로젝트가 “특별히 기술적인” 부분은 뭐가 있나요?

• 한 번 만든 화면을 **휴대폰 앱처럼** 쓰면서, **웹 브라우저**에서도 비슷하게 쓰려고
  만든 구조입니다. 
  (개발자들이 “Expo”, “웹 빌드”라고 부르는 부분 — Expo, React, React Native, React Native Web)

• 분석을 요청하는 주소(API)는 **인터넷에 올려 둔 서버**를 가리키게 할 수 있습니다.
  (배포 예: Vercel — Vercel, Node.js, Express, OpenAI API)

• “지식을 미리 책같이 저장해 두고 찾아서 붙인다(RAG)” 같은 고급 방식은
  **이 프로젝트에서는 쓰지 않습니다.**  
  그때그때 링크에서 글자를 가져와서 바로 분석하는 방식에 가깝습니다.
  (링크 본문 수집 — Cheerio 등 / 분석·신뢰도 점수 — OpenAI API / 벡터 DB·RAG·별도 팩트 DB 검색 없음.
   신뢰도의 ‘교차검증’은 프롬프트 규칙과 여러 번의 AI 평가 평균에 가깝고,
   외부 뉴스 DB와 자동 대조하는 기능은 아닙니다.)

• **데이터베이스(큰 장부에 영구 저장)** 를 쓸 수 있게 해 둔 자리는 있지만,
  **현재 사용자 기록의 핵심은 기기 안 저장**.
  (서버 DB 준비 — Drizzle ORM, PostgreSQL, Supabase 등 연결 가능(현재 사용 x) / 기기 기록 — AsyncStorage(기기 내부에 저장))
