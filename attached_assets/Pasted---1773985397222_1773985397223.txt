
반드시 아래의 평가 프로세스를 순서대로 수행해야 한다.
절대 직관적으로 점수를 주지 말고, 체크리스트 기반 감점 방식으로 계산하라.

-------------------------------------
[1단계] 평가 기준 (총 6개)
-------------------------------------

다음 6개 기준을 각각 100점에서 시작하여 감점 방식으로 평가한다.

1. 출처 권위도
2. 데이터 구체성
3. 논리적 완결성
4. 시점 유효성
5. 이해관계 투명성
6. 교차검증 일치도

-------------------------------------
[2단계] 체크리스트 기반 감점
-------------------------------------

각 기준마다 아래 결함이 존재하는지 판단하고 감점하라.

감점 기준:
-30: 치명적 결함
-25: 매우 높은 결함
-20: 중대한 결함
-15: 일반 결함
-10: 경미한 결함

각 항목은 복수 적용 가능하다.

-------------------------------------
[출처 권위도 체크리스트]
-------------------------------------
- 실명/법인 확인 불가
- 전문성 없음
- 신규 계정
- 허위정보 이력
- 개인 의견 중심

-------------------------------------
[데이터 구체성 체크리스트]
-------------------------------------
- 핵심 수치 없음
- 시각 자료 없음
- 비교 대상 없음
- 출처 없음
- 분석 없음

-------------------------------------
[논리적 완결성 체크리스트]
-------------------------------------
- 인과관계 부족
- 단정적 표현
- 사후 해석
- 리스크 미언급
- 단일 근거 일반화

-------------------------------------
[시점 유효성 체크리스트]
-------------------------------------
- 이미 반영된 정보
- 이벤트 종료
- 오래된 정보
- 과거 재가공
- 작성 시점 불명확

-------------------------------------
[이해관계 투명성 체크리스트]
-------------------------------------
- 외부 유입 유도
- 이해관계 미공개
- 조건형 콘텐츠
- 일방적 홍보
- 낚시성 콘텐츠

-------------------------------------
[교차검증 일치도 체크리스트]
-------------------------------------
- 외부 데이터 불일치
- 출처 간 충돌
- 논리 충돌
- 시점 불일치
- 비정상 구조

-------------------------------------
[3단계] 기준별 점수 계산
-------------------------------------

각 기준은 100점에서 감점 적용 후 0~100 사이 값으로 산출한다.

-------------------------------------
[4단계] 가중치 적용 (ROC 방식)
-------------------------------------

다음 가중치를 적용하여 최종 점수를 계산한다.

출처 권위도: 0.408
시점 유효성: 0.242
논리적 완결성: 0.158
이해관계 투명성: 0.103
데이터 구체성: 0.061
교차검증 일치도: 0.028

최종 점수 = Σ(기준 점수 × 가중치)

-------------------------------------
[5단계] 3/6 Rule 적용
-------------------------------------

각 기준 점수 중 40점 이상인 항목 개수를 계산한다.

조건:
- 40점 이상 기준이 4개 미만이면 → "경고 상태"

-------------------------------------
[6단계] 출력 형식
-------------------------------------

반드시 아래 JSON 형식으로 출력하라.

{
  "criteria_scores": {
    "출처 권위도": 점수,
    "데이터 구체성": 점수,
    "논리적 완결성": 점수,
    "시점 유효성": 점수,
    "이해관계 투명성": 점수,
    "교차검증 일치도": 점수
  },
  "final_score": 최종점수 (0~100),
  "passed_criteria_count": 40점 이상 개수,
  "warning": true 또는 false,
  "verdict": "신뢰 가능" 또는 "신뢰 어려움",
  "summary": "핵심 문제를 2~3줄로 요약"
}

-------------------------------------
[중요 규칙]
-------------------------------------

- 반드시 체크리스트 기반으로 판단할 것
- 근거 없이 점수 부여 금지
- 감점 이유를 내부적으로 판단하고 결과에 반영할 것
- 과도한 설명 금지 (요약만 출력)
- 항상 JSON 형식 유지
